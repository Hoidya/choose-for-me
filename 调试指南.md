# 🔍 白色页面调试指南

## 可能的原因

1. **JavaScript 运行时错误**
2. **CSS 未加载**
3. **浏览器控制台错误**
4. **组件渲染错误**

## 调试步骤

### 1. 检查浏览器控制台

打开浏览器开发者工具（F12 或 Cmd+Option+I），查看：

- **Console 标签** - 是否有红色错误信息？
- **Network 标签** - CSS/JS 文件是否成功加载？
- **Elements 标签** - HTML 结构是否正确？

### 2. 检查终端输出

查看运行 `npm run dev` 的终端，是否有：
- 编译错误
- 警告信息
- 端口占用问题

### 3. 快速测试

在浏览器控制台运行：

```javascript
// 检查 React 是否加载
console.log(window.React)

// 检查页面内容
console.log(document.body.innerHTML)
```

## 常见问题解决

### 问题 1: 控制台显示 "Module not found"

**解决：**
```bash
# 重新安装依赖
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### 问题 2: CSS 样式未应用

**检查：**
- 确认 `app/globals.css` 存在
- 确认 `tailwind.config.js` 配置正确
- 检查浏览器 Network 标签，CSS 文件是否加载

**解决：**
```bash
# 清理 Next.js 缓存
rm -rf .next
npm run dev
```

### 问题 3: 组件渲染错误

**检查：**
- 浏览器控制台的错误信息
- 终端中的编译错误

**临时解决：**
创建一个最简单的测试页面：

```tsx
// app/test/page.tsx
export default function Test() {
  return <div>测试页面</div>
}
```

访问 http://localhost:3000/test 看是否能显示

### 问题 4: 端口冲突

**解决：**
```bash
# 使用其他端口
npm run dev -- -p 3001
```

## 快速修复命令

```bash
# 1. 停止开发服务器 (Ctrl+C)

# 2. 清理缓存和依赖
rm -rf .next node_modules package-lock.json

# 3. 重新安装
npm install

# 4. 清理 Next.js 缓存
rm -rf .next

# 5. 重新启动
npm run dev
```

## 如果仍然无法解决

请提供以下信息：

1. **浏览器控制台的错误信息**（截图或复制文本）
2. **终端中的错误信息**（如果有）
3. **Network 标签中的状态**（哪些文件加载失败）

这样我可以更准确地帮你解决问题。
